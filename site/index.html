Frequently Asked Questions

What is MCP and why does it need security scanning?
Model Context Protocol (MCP) is a standard for connecting AI models (like Claude, GPT) to external tools and data sources. Each MCP server is essentially an API that an AI model can call — making it a potential attack surface. Misconfigured servers can expose filesystems, leak credentials, or allow command injection. MCP Audit audits these configurations against the OWASP MCP Top 10 security standard.
Is MCP Audit free?
Yes. The CLI is free and open source under AGPL-3.0, forever. You get all 19 security rules, multiple output formats, and CI/CD integration at no cost.
What MCP configurations does it scan?
MCP Audit automatically detects and scans claude_desktop_config.json (Claude Desktop), .mcp.json (standard MCP config), and any custom config file you point it at. It works with both stdio-based and HTTP-based MCP servers.
Can I use it in CI/CD?
Absolutely. MCP Audit ships with a GitHub Action, produces SARIF output for GitHub Code Scanning, and uses exit codes (0 = clean, 1 = critical/high findings, 2 = medium/low findings) for pipeline integration. Add it to any CI/CD system that runs CLI tools.
How is this different from a general SAST tool?
General SAST tools scan source code for bugs. MCP Audit is purpose-built for MCP server configurations — it understands the MCP protocol, knows the specific attack vectors (tool poisoning, server spoofing, credential leakage), and aligns with the OWASP MCP Top 10 standard. Think of it as "npm audit" specifically for MCP: the right tool for a specific, growing problem.
